# Quick Reference - Skills Generation System

## 11 Required JSON Fields

Every skill generation MUST include these fields:

```json
{
  "name": "kebab-case-name",           // Unique skill identifier
  "description": "One-line summary",   // What it does
  "language": "python",                // Target language
  "main_file": "main.py",              // Main source filename
  "main_code": "full source code",     // Complete implementation
  "dependency_file": "requirements.txt", // Dependency manifest
  "dependency_content": "content here", // Full dependency content
  "readme": "# README\n\nUsage...",    // Documentation
  "example_file": "example.py",        // Example filename
  "example_code": "example code",      // Working example
  "metadata": {
    "tags": ["tag1", "tag2"],
    "version": "1.0.0",
    "author": "Generated by Skills Generator"
  }
}
```

## Parser Validation Logic

```javascript
// 1. Extract JSON (markdown block or direct)
// 2. Parse JSON
// 3. Validate with validateSkillJSON(parsed, REQUIRED_FIELDS)
// 4. Check: validation.valid === true
// 5. If true → use parsed object
// 6. If false → log validation.missing fields, show error
```

### Field Validation Rules

| Field | Type | Required | Validation |
|-------|------|----------|-----------|
| name | string | YES | Must be present, non-empty |
| description | string | YES | Must be present, non-empty |
| language | string | YES | One of: python, nodejs, go, rust, java, cpp |
| main_file | string | YES | Filename only (e.g., main.py) |
| main_code | string | YES | Complete, runnable code |
| dependency_file | string | YES | Filename only (e.g., requirements.txt) |
| dependency_content | string | YES | Full file content (can be empty) |
| readme | string | YES | Full README.md content |
| example_file | string | YES | Filename only (e.g., example.py) |
| example_code | string | YES | Working example code |
| metadata | object | YES | Must contain tags, version, author |
| test_file | string | NO | Optional test filename |
| test_code | string | NO | Optional test code |

## System Prompt Key Points

### For All Languages
- Output ONLY JSON
- NO markdown code blocks (```)
- NO backticks
- NO explanatory text before/after
- Start with { and end with }
- All strings properly JSON-escaped

### Python Specifics
```
- Python 3.8+ with type hints
- PEP 8 compliance
- Google format docstrings
- requirements.txt with versions
```

### Node.js Specifics

```

### Rust Specifics

## Common Issues & Solutions

### Issue: "Missing required fields"
**Solution:** Check browser console for which fields are missing
```
[PARSER] ❌ Missing required fields: ['metadata', 'test_file']
```
The LLM response is missing these fields. Check the full response in console logs.

### Issue: "Failed to parse JSON"
**Solution:** Response doesn't have valid JSON
```
[PARSER] Response preview (first 500 chars): "Here is your skill: {"
[PARSER] Response preview (last 500 chars): "...}"
```
Look for malformed JSON syntax in the response.

### Issue: "No JSON braces found"
**Solution:** Response has no { } at all
Check if the LLM actually generated JSON or just text.

### Issue: Parser finds markdown block but validation fails
**Solution:** Markdown block is missing some fields
```
[PARSER] ⚠️ Markdown JSON missing fields: ['main_file']
```
Falls back to direct JSON extraction. If still fails, report error.

## Function Reference

### parseSkillOutput(text)
Extracts and validates JSON from LLM response
- Input: String from LLM
- Output: Parsed object or null
- Side effect: Logs [PARSER] messages to console

### validateSkillJSON(obj, requiredFields)
Validates that all required fields are present
- Input: Object, array of field names
- Output: {valid, missing, invalid, present}
- Usage: Called by parseSkillOutput()

### useSkillGenerator()
React hook for skill generation
- State: skill, loading, error
- Methods: generate(), reset()
- Providers: anthropic, gemini, openrouter
