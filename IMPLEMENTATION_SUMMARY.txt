================================================================================
API INTEGRATION
================================================================================

PROVIDERS SUPPORTED:
✓ Anthropic
  - Endpoint: https://api.anthropic.com/v1/messages
  - Authentication: x-api-key header
  - Models: claude-3-7-sonnet-20250219, etc.

✓ Google Gemini
  - Endpoint: https://generativelanguage.googleapis.com/v1beta/
  - Authentication: API key parameter
  - Models: gemini-2.5-flash, gemini-2.5-pro, etc.

✓ OpenRouter
  - Endpoint: https://openrouter.ai/api/v1/chat/completions
  - Authentication: Bearer token header
  - Models: supports multiple providers through OpenRouter

REQUEST STRUCTURE:
  - Model selection: Dynamic based on provider
  - Max tokens: 4000 (increased from default for larger outputs)
  - System prompt: Role-specific for each provider
  - User message: Includes user input + format specification

RESPONSE PARSING:
  - Extracts JSON from code blocks (```json```)
  - Falls back to direct JSON parsing
  - Validates required fields: name, skillmd
  - Error messages on parse failure

================================================================================
ERROR HANDLING STRATEGY
================================================================================

API ERRORS:
  - Network failures → "Couldn't connect" message
  - Auth failures → "Check your API key" message
  - Rate limiting → Detailed error info
  - Invalid model → Specific model not available error

PARSE ERRORS:
  - Malformed JSON → "Failed to parse" with preview
  - Missing fields → Clear field requirement message
  - Fallback: Shows raw response for debugging

USER INPUT ERRORS:
  - Empty input → Button disabled
  - No API key → Warning banner
  - Invalid selection → State validation

RECOVERY:
  - Reset button clears errors and state
  - Retry is always possible
  - No permanent failures

================================================================================
KNOWN LIMITATIONS
================================================================================

1. ZIP EXPORT
   Current: Downloads as structured text archive
   Reason: Avoids external dependencies (no jszip)
   Workaround: Users can manually zip the files
   Future: Add jszip for native ZIP support

2. SYNTAX HIGHLIGHTING
   Current: Simple markdown-aware coloring
   Limitation: Basic color coding
   Future: Could add Prism.js for full highlighting

3. FILE SIZE
   Generated SKILL.md content depends on LLM output
   Max tokens: 4000 (prevents runaway responses)
   Typical size: 2-3 KB per skill

4. RATE LIMITING
   Depends on API provider limits
   No built-in rate limiting in UI
   Relies on provider-level throttling
